Overview

VERA (Verification Engine for Results & Accountability) is an MCP server that connects Claude to California education assessment data. It enables plain-English queries against CAASPP, ELPAC, and other data sources to identify achievement gaps.

Without MCP, a County Office of Education (COE) reviewer would have to download CAASPP files manually, run their own analysis in Excel or Python, and paste results somewhere for reporting. With VERA, they open Claude Desktop, type a question in plain English, and Claude calls VERA's tools automatically — returning a formatted report. No data literacy required.

Core Capability: The Type 4 Gap

H-EDU's differentiator is identifying students who speak well but write poorly — the "oral-written delta." This computation runs on CAASPP ELA Claim 2 (writing) versus ELPAC speaking scores. A large positive delta flags students who may be misclassified or receiving inappropriate interventions.

What is MCP?

MCP (Model Context Protocol) is Anthropic's open standard for connecting AI models to external data sources and tools. Released November 2024, it has been adopted by OpenAI, Google DeepMind, and Microsoft.

The Three MCP Primitives

  1. Resources — Read-only data addresses that Claude can retrieve like a URL.
    caaspp://district/36678196000000
  2. Tools — Executable functions Claude calls with parameters. VERA has five analysis tools.
    compute_oral_written_delta(district_name='Capistrano')
  3. Prompts — Reusable templates that trigger structured workflows.
    intervention_verification_brief(district_id)

Installation & Setup

Prerequisites

  • Python 3.10 or higher
  • Claude Desktop (download from claude.ai/download)
1

Create Directory Structure

Create the VERA folder:

C:\Users\[Username]\MCP\VERA\
2

Create Python Virtual Environment

cd C:\Users\[Username]\MCP\VERA
python -m venv venv
venv\Scripts\activate
3

Install FastMCP

pip install fastmcp
4

Create the Database

python setup_database.py
5

Configure Claude Desktop

Open: %APPDATA%\Claude\claude_desktop_config.json

Add the following configuration:

{
  "mcpServers": {
    "vera": {
      "command": "C:\\Users\\[Username]\\MCP\\VERA\\venv\\Scripts\\python.exe",
      "args": ["C:\\Users\\[Username]\\MCP\\VERA\\vera_mcp_server.py"]
    }
  }
}
6

Restart Claude Desktop

Fully quit Claude Desktop (right-click taskbar icon, select Quit), then reopen.

7

Test the Connection

In Claude Desktop, type: "List all districts in the VERA database"

The Five Tools

list_districts()

Lists all districts available in the VERA database.

Parameters: None

Example: "List all districts in the VERA database"

fetch_caaspp_results(district_name, grade?, subgroup?)

Fetches CAASPP assessment results for a district.

Parameters: district_name (required), grade (optional), subgroup (optional)

Example: "Fetch CAASPP results for Capistrano Unified, Grade 4"

compute_oral_written_delta(district_name, subgroup?)

H-EDU's core differentiator. Compares ELA Claim 2 (writing) against ELPAC speaking scores to identify Type 4 gaps.

Parameters: district_name (required), subgroup (optional, defaults to English Learners)

Example: "Compute the oral-written delta for Capistrano English Learners"

flag_type4_candidates(threshold?)

Scans all districts for Type 4 gaps. The statewide accountability view.

Parameters: threshold (optional, defaults to 8.0)

Example: "Flag all Type 4 candidates across all districts"

get_lcap_match_summary(district_id)

Generates an LCAP verification report for COE submission.

Parameters: district_id (required)

Example: "Generate an LCAP match-rate summary for district 36678196000000"

Resources & Prompts

MCP Resources

Resources are read-only data endpoints that Claude can access directly:

  • caaspp://districts — List all districts
  • caaspp://district/{id} — Get data for specific district

MCP Prompts

Prompts are workflow templates that chain multiple tool calls:

intervention_verification_brief(district_id)

  1. Pulls LCAP alignment data
  2. Computes oral-written deltas
  3. Summarizes findings for COE submission

statewide_gap_scan()

Generates a statewide Type 4 gap analysis grouped by county and district.

Production Deployment

Local Mode (Current)

VERA runs in stdio mode for Claude Desktop connections. No server infrastructure required.

HTTP Mode (Production)

For multi-user access, VERA can run as an HTTP server with OAuth 2.1 authentication:

# In vera_mcp_server.py, change:
mcp.run()  # stdio mode

# To:
mcp.run(transport="http", port=8000)  # HTTP mode

Database Options

  • SQLite — Current demo setup, single-file, no server
  • PostgreSQL — Production multi-user, use Supabase or AWS RDS
  • Live CDE API — Direct connection to California data sources

Security Considerations

  • OAuth 2.1 for authentication in HTTP mode
  • Data Processing Agreements (DPA) required for student-level data
  • Non-evaluation guarantee: no teacher identity attached to results
